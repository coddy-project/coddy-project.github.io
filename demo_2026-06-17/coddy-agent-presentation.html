Проблема

  • ☁️
    Облако = чужой компьютер Код улетает во внешние API. Compliance, NDA, корпоративная безопасность — несовместимо с cloud-first агентами.
  • 📦
    Тяжёлые рантаймы не влезают в distroless Claude Code = Node.js. В scratch или distroless образ не запустишь. Container fleets ждут.
  • 🔌
    Нет единого backend для всех клиентов Cursor, скрипты, CI-пайплайн, Telegram-бот — у каждого отдельная история, нет общего стейта.
  • 🦙
    On-premise модели остаются за бортом Ollama, DeepSeek, Qwen локально — нормального AI agent harness для них нет.

Решение — Coddy Agent

Один статический Go-бинарник.   35 МБ.   Без рантайма.
🐋  Distroless-ready
Работает в scratch, distroless/static, read-only rootfs. Идеально для container fleets.
🔌  ACP harness
Agent Control Protocol — протокол-первый дизайн. Один backend для Cursor, Zed, HTTP, мессенджеров.
🤖  Multi-provider LLM
OpenAI, Anthropic, Ollama, любой OpenAI-compatible API. Никакого vendor lock-in.
🔒  On-premise first
Данные не покидают ваш контур. Docker Compose — 2 строки. Или просто запустите бинарник.

Три режима работы

coddy acp
ACP · JSON-RPC 2.0 over stdio
Cursor и Zed как external agent. Streaming, permission prompts, восстановление сессий.
coddy http
OpenAI-compatible API + встроенная SPA
POST /v1/responses — любой OpenAI-клиент работает с Coddy как бэкендом. n8n, Open WebUI, ваш Python-скрипт.
coddy gateway
Telegram Bot (и другие мессенджеры)
Per-user сессии, ACL, isolation modes. Расширяемо: Discord, Slack — через Gateway adapter.
Все три режима используют одно хранилище сессий — IDE + браузер + Telegram синхронизированы автоматически
