Run a coding agent from one static binary

Coddy is a harness for software work: ReAct loop, filesystem and shell tools, MCP, project rules, skills, optional browser UI, scheduler, and long-term memory. Built for containers, IDE clients, and your own API keys.

Quick install

curl -fsSL https://coddy.dev/install.sh | bash
irm https://coddy.dev/install.ps1 | iex

Downloads the latest release binary, installs to your PATH, and creates ~/.coddy/config.yaml from the example when missing. Then set a provider key and run coddy http or coddy acp.

Docker: ghcr.io/coddy-project/coddy-agent - compose setup

What is Coddy

Most agent stacks chase a general chat assistant. Coddy targets a coding agent: Cursor-style rules and skills on disk, plan and agent modes, tool permissions, streaming UI, and sessions you can resume from an IDE, browser, or automation.

Ship it as a single binary in distroless or from-scratch images, or run a fleet of containers with your orchestrator defining CPU, memory, and mounts - one agent per task, full control of each instance.

Features

ACP

coddy acp over stdio. Sessions, modes, permissions, streaming updates for IDE clients and scripts.

HTTP

OpenAI-compatible gateway on coddy http. POST /v1/responses, models, Swagger at /docs/.

UI

Embedded SPA at / with http,ui. Chat, sessions, plan documents, scheduler drawer, settings, multimodal file attachments.

Rules

Auto-discover .cursor/rules, .coddy/rules, .claude/rules, and .codex/rules under the project cwd.

Skills

Slash commands from SKILL.md packs in skills.dirs, including ~/.cursor/skills and project .skills.

MCP

Connect MCP servers from config or per session. Tool definitions merge into the agent and plan loops.

ReAct

Reasoning, tool calls, and observations in a loop. Built-in tools plus MCP, with agent and plan modes.

Scheduler

Cron jobs with a UI drawer and agent tools for recurring tasks.

Memory

Optional long-term memory copilot and session memory routes.

Multimodal

Attach images and files in the composer when the model supports it. Assets saved to the session directory and injected into the agent context.
